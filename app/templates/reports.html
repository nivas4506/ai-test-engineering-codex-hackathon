Final report anatomy

TestDebugger reports are designed to be shown to developers, judges, and reviewers without hiding the raw evidence behind the summary.

What it contains

Execution evidence

  • Run status and iteration count
  • Planner and generation history
  • Pytest stdout and stderr
  • Artifact file paths
Why it matters

Decision-ready output

  • Separates passing runs from pipeline errors
  • Shows when the debugger intervened
  • Makes retries explainable
  • Keeps local artifacts inspectable
Current implementation

Where to inspect generated reports

Each run is stored under `workspace/runs/{run_id}`.

Start from the Overview page to trigger orchestration, then inspect the JSON panel or open the persisted `final_report.json` from the local workspace for deeper review.
