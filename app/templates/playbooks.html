Playbooks

How teams run TestDebugger

Use focused playbooks to move from a cold repository to a usable execution report without losing traceability across retries.

Playbook A

First-pass repository scan

Best for new repos with no tests.

Run analysis, inspect the planner output, generate tests once, then review whether the module inventory looks sane before scaling up.

Playbook B

Failure isolation loop

Best for brittle or side-effect-heavy code.

Use balanced mode first. If generation overreaches, let the debugger shift into safe mode and compare the logs side by side in the report view.

Playbook C

Demo-ready artifact export

Best for hackathons and reviews.

Run orchestration against a sample repo, then open the report page to present iterations, generated files, and final status as a clean narrative.
