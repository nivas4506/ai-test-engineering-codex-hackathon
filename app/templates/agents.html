Agent Stack

TestDebugger splits code understanding, generation, execution, debugging, and orchestration into focused roles so each stage can be observed and improved independently.

01

Planner

Maps modules, risk areas, and test strategy before any files are generated.

  • Repository scanning
  • Function inventory
  • Priority scoring
02

Generator

Produces test files with balanced and safe execution modes for iterative recovery.

  • Pytest file generation
  • Edge-case coverage
  • Import/bootstrap support
03

Executor

Runs real pytest commands and keeps stdout, stderr, duration, and failure metadata.

  • Real subprocess execution
  • Failure parsing
  • Artifact persistence
04

Debugger

Inspects failing runs and lowers generation aggressiveness when the system made unsafe assumptions.

05

Orchestrator

Coordinates the loop from analysis to final report and determines when to stop retrying.
